Cabinet Architecture

OSS maintainer's AI agent cabinet — 8 specialists + planner

Reactive Mode (Webhook)

  • • GitHub event → webhook handler → processEvent()
  • • Planner Agent decides which specialists to invoke
  • • Manager runs agents, persists traces, posts labels/comments/check runs
  • • All writes gated by cabinet.yml autonomy flags

Autonomous Mode (Morning Brief)

  • • Cron fires hourly → checks WatchedRepo scheduleHour
  • • Fetches issues + PRs + commits via gh CLI (public mode)
  • • Parallel triage + review across full queue
  • • Priority Agent reads RepoMemory → ranks top items
  • • Briefing Agent writes email → AgentMail delivers

Observability & Safety

  • • Every agent step persisted: input, output, tokens, cost, latency
  • • Clickable trace flowchart in Next.js UI
  • • RepoMemory persists recurring themes across briefs
  • • 20 eval cases with GitHub Action regression testing
  • • Never merges/approves/closes — defensive by design
